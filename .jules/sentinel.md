## 2024-07-16 - Pinning GitHub Actions to Commit SHAs
**Vulnerability:** The CI/CD workflow at `.github/workflows/CCCCP.yml` used mutable version tags (e.g., `@v4`) for its GitHub Action dependencies. This created a supply-chain risk, as a compromised tag could point to a malicious version of an action, injecting vulnerabilities into the build process.
**Learning:** Relying on floating tags for CI/CD dependencies is a critical security oversight. It sacrifices security for minor convenience, creating a vector for supply-chain attacks that can be easily overlooked. Verifying the exact commit SHA for a given tag and using that immutable reference is a simple yet powerful hardening technique.
**Prevention:** All GitHub Actions in CI/CD workflows must be pinned to their full, immutable commit SHAs. This should be a standard security practice for any new workflow and a required fix for any existing ones. This prevents the execution of untrusted code if an action's repository is compromised.
