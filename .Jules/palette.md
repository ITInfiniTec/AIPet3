## 2024-02-05 - Improving Documentation UX

**Learning:** When documentation acts as the primary interface of a project, applying UX principles like scannability (Table of Contents), hierarchy (proper headings), and visual feedback (progress indicators) is as critical as UI polish. However, some automated tools like `markdown-link-check` may fail to correctly slug headings that contain emojis, requiring a balance between visual delight and tool compatibility.

**Action:** Use emojis in Table of Contents for visual appeal, but keep main headings clean (or place emojis at the end) to ensure internal links remain robust across different markdown parsers and link checkers.
